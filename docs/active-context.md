# Active Context — chore/expo-go-compat: Upgrade Expo SDK 52 → SDK 54

## Context

The Expo Go app on-device (both iOS and Android) requires SDK 54, but the project targeted SDK 52.
This caused a "SDK mismatch" error preventing the app from loading. The fix is a dependency-only
upgrade: bump `expo` and all Expo-peer packages to their SDK 54-compatible versions, update
`app.json` sdkVersion to `54.0.0`, and bump `jest-expo` to match. No source code changes required —
all SDK 54 APIs used by this project are backwards-compatible.

Branch: `chore/expo-go-compat`

## User Action Required

After the upgrade, create `mobile/.env.local` (gitignored, never committed) if not already present:

```
EXPO_PUBLIC_SUPABASE_URL=<Project URL from Supabase Dashboard → Project Settings → API>
EXPO_PUBLIC_SUPABASE_ANON_KEY=<anon/public key — starts with eyJ>
```

Then run:

```bash
cd mobile && npx expo start --tunnel
```

Scan QR with Expo Go SDK 54. Expected: app loads, no SDK mismatch error.
